### 🔴 Tier 2 — Code 결함 (Codex review 실측)

**P8. Task 1.8 atomic-write — parent dir fsync 순서 잘못** [Codex rev #3]
- Plan line 1318-1321.
- 문제: 현재 코드는 (1) fsync(tmp file) → (2) fsync(parent dir) → (3) rename. 하지만 rename 결과를 보장하려면 (3) rename 후 (2) parent dir fsync 가 필요. crash window 존재.
- Fix: 순서 → fsync(tmp file) → rename → fsync(parent dir) → readback validate.

**P9. Task 1.7 lock.js — stale 검출 throw 가 catch 에서 swallow** [Codex rev #4]
- Plan line 1238-1243.
- 문제: stale lock 발견 시 throw 가 outer catch 의 generic backoff retry 안에 갇힘 — 사용자 안내 ("run --unlock") 가 표시 안 됨.
- Fix: typed error (`STALE_LOCK` code) 도입, outer catch 가 typed error 만 swallow 하고 stale 은 propagate.

**P10. Task 1.3 envelope — `ulidLike()` vs test regex mismatch** [Codex rev #5]
- Plan line 821 (test: `/^[a-z0-9]+$/`) vs 843-845 (impl: `Date.now() + '_' + randomBytes`).
- 문제: `_` 가 들어가서 regex 매치 실패. `node --test` 즉시 FAIL.
- Fix: test regex 를 `/^[a-z0-9_]+$/` 또는 impl 에서 `_` 제거.

**P11. Task 1.1/1.3 M3 envelope provenance 확장 — suite schema 충돌 가능** [Codex rev #6]
- Plan line 546-575 (memory-card schema 의 source_artifacts) + 858-872 (envelope.wrap 호출).
- 문제: deep-memory 가 source_artifacts 에 `id`/`content_hash`/`captured_at`/`artifact_kind`/`schema_version` 추가했는데, suite M3 envelope schema 가 `additionalProperties: false` 라면 sibling validator 가 reject.
- Fix: deep-suite/schemas/artifact-envelope.schema.json 확인 후 — (a) 추가 필드를 `x-deep-memory-*` extension namespace 로 옮김 OR (b) payload 안으로 옮김 OR (c) suite schema 가 `additionalProperties: true` 면 OK.

**P12. Task 4.2 fts-index upsertCard input shape mismatch** [Codex rev #8]
- Plan line 2499-2502 (test: top-level `memory_id`/`memory_type`/`privacy_level`) vs 2528-2534 (impl: `card.payload.*`).
- 문제: test fixture 가 wrapped card shape 가 아니고 flat object — impl 이 `card.payload.memory_id` 읽으면 undefined.
- Fix: test fixture 를 `{ payload: {...}, project_id: '...' }` 로 정렬 또는 impl 이 두 shape 모두 accept.

**P13. Task 1.10 + 4.2 BM25 ranking 방향 뒤집힘 (silent quality bug)** [Codex rev #9]
- Plan line 1441-1444 (score.js: 큰 raw bm25 → 1.0 normalize) vs 2544-2548 (fts-index: `ORDER BY bm25 LIMIT`).
- 문제: SQLite FTS5 의 `bm25()` 는 **낮은 값이 better match** (negative log). `bm25MinMax` 가 큰 값을 1.0 으로 정규화하면 worst match 를 best 로 ranking.
- Fix: `bm25MinMax` 를 `(max - raw) / (max - min)` 로 invert. 또는 raw 를 `-bm25` 로 변환 후 min-max.

**P14. Task 1.12 llm-bridge adapter name path traversal** [Codex rev #10]
- Plan line 1558-1563, 1581-1583.
- 문제: `require('./adapters/' + chosen)` — chosen 이 user-controlled (config.yaml) 일 수 있어 `../../../etc/passwd` 같은 traversal 가능.
- Fix: allowlist map `const ADAPTERS = { 'claude-agent': require('./adapters/claude-agent'), 'codex-bash': require('./adapters/codex-bash'), ... }; const mod = ADAPTERS[chosen];`.
